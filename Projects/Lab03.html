Module 2 Reflection: Semantic HTML (H2)

Published October 23, 2025

Structuring Content Correctly (H3)

This lab confirmed that HTML is all about adding the proper meaning to content, not about how it looks. Using structural tags like <header>, <main>, <article>, and <footer> ensures the document's hierarchy is clear to search engines and accessibility tools.

The core takeaway was remembering the appropriate use of list tags and headings, especially applying the correct semantic structural tags like <header> and <footer> rather than simply relying on <div> tags.

For future reference, here are the required structural elements:

  1. Use a consistent heading structure (H1, H2, H3).
  2. Always wrap the primary content in the <main> tag.
  3. Ensure stand-alone content is inside an <article> tag.

Essential tools for this process included:

  • The W3C Validator, which is essential for catching syntax errors.
  • The MDN Web Docs for tag definitions.
